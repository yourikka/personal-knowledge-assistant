# UI 交互约定

这份文档记录个人知识库 Web 工作台当前已经稳定的前端交互规则，后续调整页面时默认遵守，除非产品逻辑明确变化。

## 文档内容区

- `文档内容` 区块展示的是多篇文章的卡片列表，不是单篇正文阅读器。
- 每张卡片只展示概览信息：
  - 标题
  - 摘要或关键内容
  - 标签
  - 分类
  - 置信度
  - 来源类型
  - 关联数量
- 这个区块必须是固定高度并独立滚动，避免文档数量增多后把整页撑得过长。
- 在主页面里，不直接展示文章全文。

## 阅读页

- 点击 `文档内容` 区块中的任意文章卡片，打开居中的阅读页。
- 阅读页才展示完整正文。
- 阅读页内部必须独立滚动，不能带动背景页面滚动。
- 阅读页保留正文视图切换：
  - `清洗后`
  - `原始内容`
- 阅读页支持关闭方式：
  - 点击关闭按钮
  - 点击遮罩空白处
  - 按 `Esc`

## 搜索行为

- `文档内容` 区块顶部的搜索框默认用于筛选文章卡片列表。
- 如果阅读页已经打开，搜索词同时用于高亮阅读页中的正文匹配内容。
- 主页面不因为搜索而展开全文，只改变卡片列表结果。

## 列表与详情的职责边界

- 文档列表接口保持轻量，适合驱动主页面卡片列表。
- 文档详情接口返回全文、清洗后文本和元数据，适合驱动阅读页。
- 主页面概览和阅读页全文是两个不同层次，不要混在同一个区块里。
